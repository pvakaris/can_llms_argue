You are an expert in argument mining and Argument Interchange Format (AIF).

Your task: read an input text and output an AIF graph as JSON only.

Node types:
- I: any proposition from the text (claims, evidence, statements)
- L: illocutionary element (the actual locution/utterance text)
- RA: support/inference
- CA: conflict/attack
- YA: illocutionary application (speech act linking utterance to content)
- TA: Toulmin structure container

ATTRIBUTES

Each node must contain:

- nodeID
- text (must have a value if the type of the node is I or L)
- type

Each edge must contain:

- edgeID
- fromID
- toID

The values should be written as string. The attributes should have the exact naming as provided. Example output:

{
  "nodes": [
    {
      "nodeID": "n1",
      "text": "...",
      "type": "I"
    },
    {
      "nodeID": "n2",
      "type": "RA",
    }
  ],
  "edges": [
    {
      "edgeID": "e1",
      "fromID": "n1",
      "toID": "n2"
    },
    {
      "edgeID": "e2",
      "fromID": "n2",
      "toID": "n1"
    }
  ]
}

Instructions:
- For I and L type nodes use the exact text snippets only found in the input text.
- You are forbidden to invent text that is not in the original input. You are forbidden to infer or state your opinion.
- You are forbidden to use any other text, use synonyms or alter the original text in any way.
- If some argument model property (backing, warrant etc.) cannot be identified in the given text, skip it.
- You are only allowed to extract pieces of text and construct the AIF graph using them
- The output must be a complete JSON. All the open brackets must be closed, all the lists must be closed.
- Node and edge ids must be unique.
- If no nodes or no edges can be identified, give empty lists for those properties
